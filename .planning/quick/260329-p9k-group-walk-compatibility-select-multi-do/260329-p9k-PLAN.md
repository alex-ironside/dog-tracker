<objective>
Add group-context walk logging and graph visualization. Users can log walks with two groups where compatibility is context-dependent (e.g., A and C are fine alone, but when B joins, A+B attack C). The graph shows these group-context conflicts as hyperedge nodes.

Purpose: Enable behaviorists to capture nuanced group dynamics where pair compatibility changes based on which other dogs are present.
Output: Updated walk log sheet with group mode, walk history display with group info, graph with hyperedge nodes for group-context conflicts.
</objective>

**2. Scoring helper for group-context inference (src/lib/scoring.ts):**

Add a new exported function:

```typescript
export function inferGroupContextConflicts(
  walkHistory: WalkLogEntry[]
): { triggerIds: string[]; targetId: string; status: CompatibilityStatus }[]
```

Logic:
- Scan walk history for entries that have `groupContext` AND `pairOutcomes` containing an 'incident' or 'poor' outcome for a cross-group pair
- For each such cross-group conflict pair (dog from groupA vs dog from groupB), record the "trigger group" (the group the aggressor-side dogs are in) and the target dog
- Group these by pattern: if multiple entries show the same set of dogs in groupA causing issues with a dog in groupB, emit a single hyperedge record: `{ triggerIds: [A, B], targetId: C, status: 'conflict' }`
- Only emit hyperedges where the trigger group has 2+ dogs (otherwise it's just a regular pair edge, no hyperedge needed)
- Deduplicate: if the same triggerIds+targetId appears multiple times, keep the worst status

**3. Graph hyperedge rendering (src/components/CompatibilityGraph.tsx):**

Add hyperedge visualization for group-context conflicts:

- Call `inferGroupContextConflicts(walkHistory)` in the `graphData` useMemo
- For each hyperedge `{ triggerIds: [A, B], targetId: C }`:
  - Create a synthetic "group node" with id like `group_A|B` and a label like "A, B" (using dog names). Give it a distinct visual: smaller node, diamond shape or different color (use `#f97316` orange fill, smaller radius 4)
  - Create links: one link from each trigger dog to the group node (thin, dashed, gray `#94a3b8`), and one link from the group node to the target dog (thick, red `#ef4444`, solid)
- In `nodeCanvasObject`, detect group nodes by id prefix `group_` and render them as a small diamond (rotated square) with orange fill instead of the standard blue circle. Label them with the joined trigger dog names.
- In `nodePointerAreaPaint`, also handle group nodes with the diamond hit area.
- Ensure the group node does not appear in the `handleNodeClick` handler (clicking it should be a no-op, not open DogPanel).

The key insight: regular pair edges still show as before. Hyperedge nodes ONLY appear when walk history reveals group-context-dependent conflicts that differ from the pair-level relationship.
