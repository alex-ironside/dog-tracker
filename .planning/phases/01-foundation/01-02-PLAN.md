**3. `src/components/DogPanel.tsx`** — Per UI-SPEC section 3.4:

Props: `open: boolean`, `onOpenChange: (open: boolean) => void`, `editingDog: Dog | null` (null = add mode)

Uses shadcn `<Sheet side="right">` wrapping `<SheetContent className="w-full max-w-md">`.

Panel header: title "Add Dog" or "Edit Dog" based on mode. Close button with `<X size={18} />` and `aria-label="Close panel"`.

Form fields (vertical stack, `space-y-4`):
- Name: `<Input placeholder="e.g. Rex" />` — required
- Breed: `<Input placeholder="e.g. Labrador" />` — optional
- Age: `<Input type="number" min={0} placeholder="e.g. 3" />` — optional, integer
- Notes: `<textarea rows={3} placeholder="Any notes about this dog..." className="..." />` (style like shadcn Input) — optional

Each field wrapped with `<Label>` above using `htmlFor`.

Validation: on submit only (not on blur). If name is empty: show `<p className="text-sm text-red-600 mt-1">Name is required.</p>` below the Name field and set Name input border to `border-red-500 focus-visible:ring-red-500`.

Footer (sticky): `<div className="sticky bottom-0 bg-white border-t border-slate-200 py-4 px-6 flex justify-between">`:
- Discard: `<Button variant="outline">Discard</Button>` — closes panel, resets form
- Save Dog: `<Button variant="default">Save Dog</Button>` — validates and calls addDog or updateDog

Use `useState` for form state. When `editingDog` changes (or panel opens), populate form fields from the dog's current data (edit mode) or clear them (add mode).

On save in add mode: call `useAppStore.getState().addDog({ name, breed, age: age ? parseInt(age) : null, notes })`. On save in edit mode: call `useAppStore.getState().updateDog(editingDog.id, { name, breed, age: age ? parseInt(age) : null, notes })`. Close panel after save.
