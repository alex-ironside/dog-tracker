### CSS Grid Implementation

WeekCalendar uses `display: grid` with:
- 8 columns: `grid-cols-[48px_repeat(7,1fr)]` (first col = hour labels, 7 = day columns)
- 13 rows: `grid-rows-[auto_repeat(12,minmax(48px,auto))]` (first row = day headers, 12 = hours 07–18)
- `border-collapse` effect via `border border-slate-100` on each cell

Hour range: 07:00–18:00 inclusive (12 rows). Source: ROADMAP.md states "07:00–19:00" — this renders 12 one-hour slots covering 07:00, 08:00, … 18:00 (i.e., the last slot is 18:00–19:00).

## Interaction Contracts

### Drag-and-Drop

| Interaction | Trigger | Result |
|-------------|---------|--------|
| Drag group from sidebar | `PointerSensor` (distance: 8 activation) + `KeyboardSensor` | DragOverlay shows group name card `opacity-70` |
| Drop onto empty slot | `onDragEnd` with valid `over.id` | Group scheduled in slot; removed from sidebar |
| Drop onto occupied slot | `onDragEnd` with occupied `over.id` | Drop rejected — no state change; slot shows `ring-red-200` flash (100ms CSS transition) |
| Drop group with existing slot | Drop onto different empty slot | Previous slot cleared; group scheduled in new slot (D-08) |
| × button on slot card | Click | Group unscheduled; returns to sidebar as draggable |
| Drag scheduled card back to sidebar | `onDragEnd` with `over.id === 'group-sidebar'` | Group unscheduled; sidebar accepts it (D-09) |

### Week Navigation

| Interaction | Trigger | Result |
|-------------|---------|--------|
| View next week | Click ChevronRight | `weekOffset` local state +1; column headers update to next week dates; schedule slots re-evaluate against new week |
| View previous week | Click ChevronLeft | `weekOffset` local state -1 |
| Default on mount | Component mount | `weekOffset = 0` (current calendar week) — not persisted (D-04, deferred) |

### Accessibility

- Tab bar button: `role="tab" aria-selected={...}` — matches existing App.tsx pattern
- Calendar grid: `role="grid"` on `WeekCalendar`, `role="gridcell"` on each `CalendarSlot`
- Droppable slots: `aria-label="[DayName] [Hour]:00 — empty"` or `"[DayName] [Hour]:00 — [GroupName]"`
- Remove button: `aria-label="Remove [GroupName] from [DayName] [Hour]:00"`
- Keyboard drag support: `KeyboardSensor` from dnd-kit (matches Phase 4 pattern)

## States and Visual Feedback

| State | Component | Visual Treatment |
|-------|-----------|-----------------|
| Empty slot (idle) | `CalendarSlot` | No border, transparent background |
| Empty slot (drag-over, accepts) | `CalendarSlot` | `ring-2 ring-primary ring-inset` |
| Occupied slot (idle) | `CalendarSlot` | Contains `ScheduledGroupCard` |
| Occupied slot (drag-over, rejects) | `CalendarSlot` | `ring-2 ring-red-300 ring-inset opacity-50` |
| Sidebar panel (drag-over) | `GroupSidebar` | `bg-slate-100` (matches RosterPanel in GroupBuilder) |
| Drag in flight | `DragOverlay` | `opacity-70 shadow-lg` ghost card |
| Group with conflicts | `ScheduledGroupCard` | Amber ⚠ `AlertTriangle` icon inline after dog count |
| Score badge (high ≥ 80) | `ScheduledGroupCard` header if shown | `bg-green-100 text-green-700` |
| Score badge (mid 50–79) | `ScheduledGroupCard` header if shown | `bg-yellow-100 text-yellow-700` |
| Score badge (low < 50) | `ScheduledGroupCard` header if shown | `bg-red-100 text-red-700` |

Note on score badge in slot cards: Slot cards (inside the grid) are compact. Score badge is omitted from `ScheduledGroupCard` by default to keep slots minimal. Score is visible in the sidebar card or group panel in the Groups tab. This is a discretionary simplification — the slot card shows only name + dog count + conflict icon.
