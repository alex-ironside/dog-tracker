## Interaction Contracts

### Log a walk — 3 entry points, all open WalkLogSheet

| Entry Point | Pre-fill behaviour | Sheet title |
|-------------|-------------------|-------------|
| History tab "Log a walk" button | No pre-fill; date=today; dogs=none selected | "Log a Walk" |
| CalendarSlot "Log" button (on ScheduledGroupCard) | `initialDogIds` = group's current dogIds; `initialDate` = today; `initialGroupId` = groupId | "Log Walk — [Group Name]" |
| DogPanel History tab "Log a walk for [name]" button | `initialDogIds` = [current dog's id]; date=today | "Log Walk for [Dog Name]" |

### Outcome selection (in WalkLogSheet)

- 5 toggle buttons in a row: Great, Good, Neutral, Poor, Incident
- Only one can be selected at a time (aria-pressed pattern, matching EdgeSheet status picker)
- Selected state: `ring-2 ring-offset-1 ring-slate-500` (matches EdgeSheet selected button pattern)
- No outcome selected = Save button disabled
- Outcome button text colours: inherit from OutcomeBadge semantic palette (Great=green-700, etc.)

### Dog multi-select (in WalkLogSheet)

- Checkbox list of all active dogs
- Scrollable if list exceeds 5 dogs (`max-h-48 overflow-y-auto`)
- Pre-selected dogs (from props) are checked on open
- At least 1 dog required; validation error if Save pressed with 0 dogs

### Save Walk Log

- Validation before save: outcome required + at least 1 dog selected
- On validation failure: show inline error text (text-sm text-red-600 role="alert") beneath the failing field
- On save success: Sheet closes; entry appears at top of History tab list; no toast (consistent with existing app — no toast library installed)
- Entry is immutable after save — no edit affordance exposed anywhere

### "Log" button on ScheduledGroupCard

- Placed after the group name/dog count, before the X remove button
- Button: text-xs, variant="ghost", `text-slate-500 hover:text-slate-700`, label "Log"
- Does NOT navigate to History tab — opens WalkLogSheet as a Sheet portal within the CalendarScheduler context (consistent with EdgeSheet pattern, source: RESEARCH.md open question resolution)

### DogPanel History tab

- Default tab on open: "Profile" (preserving existing behaviour for add/edit flows)
- History tab shows chart first, then "Log a walk for [name]" button, then recent entries list (last 10, no pagination for v1)
- Chart empty state (no entries for this dog): render chart container area with centred text "No walks logged yet"

## Accessibility Contracts

- All Sheet components use `SheetTitle` (Radix DialogTitle) — required for screen reader announcement (established in Phase 1)
- Outcome toggle buttons use `aria-pressed` to communicate selection state (established pattern from EdgeSheet status picker)
- Tab switcher inside DogPanel uses `role="tab"` and `aria-selected` (matches App.tsx tab bar pattern)
- Dog multi-select checkboxes: each row is a `<label>` wrapping `<input type="checkbox">` with dog name as visible label text
- Form validation errors: `role="alert"` and `aria-live="polite"` (matches DogPanel nameError pattern)
- CalendarSlot "Log" button: `aria-label="Log walk for [group name] at [day] [hour]:00"` (follows existing ScheduledGroupCard aria-label pattern)
- Chart (WalkHistoryChart): `aria-label="Walk outcome history for [dog name]"` on the ResponsiveContainer wrapper `<div>`

## Phase-Specific Notes

### Recharts version lock

Use recharts@3.8.1 (npm latest as of 2026-03-28). Recharts 3.x breaking change that affects this phase: `data` prop goes on `<Scatter data={...}>`, NOT on `<ScatterChart>`. XAxis requires `type="number"` for ms-timestamp X values. Source: RESEARCH.md.

### No new shadcn components needed

All required UI primitives (Sheet, Button, Input, Label) are already installed. The log form's date input uses a native `<input type="date">` styled with the same textarea/input CSS class pattern from DogPanel — no datepicker component needed for v1.

### No toast / notification on save

The existing app has no toast library. On successful save, the Sheet closes and the new entry appears at the top of the History tab list. No toast, snackbar, or notification is shown — consistent with all prior phases.

### DogPanel tab: simple button toggle, not Radix Tabs

Radix Tabs is not installed. Phase 6 uses the same `role="tab"` / `aria-selected` button pattern as App.tsx. This avoids adding a new Radix dependency for two tabs. Source: RESEARCH.md Pattern 5 + CONTEXT.md discretion area.

### Filtering in History tab

v1: no filtering. The History tab shows all entries in reverse-chronological order. No filter UI is rendered. Source: CONTEXT.md Deferred Ideas.
