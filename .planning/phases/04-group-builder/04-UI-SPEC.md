## Layout Contract

Two-panel layout inside the Groups tab panel:

```
┌─────────────────────────────────────────────────┐
│  Tab bar: Dogs | Compatibility | Groups          │
├──────────────────┬──────────────────────────────┤
│  Roster panel    │  Groups panel                │
│  (left, ~280px)  │  (right, flex-1)             │
│                  │                              │
│  Compact list    │  Groups stacked vertically   │
│  of active dogs  │  each group is a card        │
│  (drag handles)  │  + Add Group button (top)    │
│                  │                              │
└──────────────────┴──────────────────────────────┘
```

- Left panel (roster): fixed or min-w-[280px], overflow-y-auto, bg-slate-50, border-r border-slate-200
- Right panel (groups): flex-1, overflow-y-auto, bg-white, p-4 gap-4 flex flex-col
- Outer DndContext wraps both panels
- No horizontal scroll on the groups panel — groups stack vertically

### Roster Row
- `flex items-center gap-2 px-3 py-2 rounded-md`
- GripVertical icon (16px, text-slate-300, cursor-grab) — left
- Dog name (text-sm text-slate-900) — flex-1
- Assignment label when in-group: "in [Group Name]" (text-xs text-slate-400 italic) — right side
- In-group state: `opacity-60 cursor-not-allowed` — non-draggable
- Available state: `hover:bg-slate-100` — draggable

### Group Panel Card
- `border border-slate-200 rounded-xl bg-white shadow-sm`
- Header: `flex items-center justify-between px-4 py-3 border-b border-slate-200 bg-slate-50 rounded-t-xl`
  - Group name: inline-editable input (click to edit, blur/Enter to commit; text-sm font-semibold text-slate-900)
  - Score badge: numeric 0–100, styled as `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium` — color mapped by score: ≥80 green-100/green-700, 50–79 yellow-100/yellow-700, <50 red-100/red-700
  - Warning icon: `⚠` (AlertTriangle from lucide-react, size 14, text-amber-500) — shown only when conflicts exist in the group
  - Delete button: Button variant="ghost" size="icon" (Trash2 icon, size 14, text-destructive) — far right
- Body: `px-4 py-3 min-h-[64px]`
  - Mini dog cards in a horizontal flex-wrap row, gap-2
  - Drop zone hint when empty: dashed border `border-2 border-dashed border-slate-200 rounded-lg`

### Mini Dog Card (inside a group)
- `inline-flex items-center gap-1 px-2 py-1 rounded-md bg-slate-100 border border-slate-200 text-sm text-slate-900`
- Dog name — text-sm text-slate-900
- × remove button — Button variant="ghost" size="icon" (X icon, size 12, text-slate-400 hover:text-red-500), p-0 h-4 w-4

### SVG Conflict Overlay
- Absolute-positioned SVG layer over the group body (same bounding rect as the mini cards container)
- Each conflicting pair: a `<line>` element, `stroke="#ef4444"` (red-500), `strokeWidth={1.5}`, `opacity={0.7}`
- Lines connect the center points of the two conflicting mini dog cards
- Clicking a conflict line opens the existing `EdgeSheet` component with the two dog names and current status
- The SVG overlay does not intercept non-conflict pointer events (pointer-events on non-line elements = none)

Source: CONTEXT.md D-02, D-03, D-04; DogCard.tsx for card visual baseline; CompatBadge.tsx for badge pattern

## Interaction Contract

### Drag and Drop (dnd-kit)
- Sensors: PointerSensor (activation constraint: distance 8px to avoid accidental drag on click), KeyboardSensor
- DragOverlay: shows a semi-transparent copy of the roster row while dragging (opacity-70, shadow-lg)
- Drag state committed only in `onDragEnd` — no optimistic UI during drag (per CONTEXT.md D-16)
- Invalid drops (same group, already-in-group dog) produce no state change and no error; the card snaps back

### Inline Group Name Edit
- Click on group name text: replaces with `<input>` (same font/size, autoFocus, select-all on mount)
- Blur or Enter: commits the value if non-empty; reverts to previous name if empty
- Escape: cancels edit, reverts to previous name
- Input width: min-w-[120px] max-w-[240px]

### Group Score Update
- Score re-calculates on every `onDragEnd` (dog added/removed) using `scoreGroup` from `src/lib/scoring.ts`
- Score badge color updates immediately — no loading state needed (synchronous pure function)

### Delete Group
- Clicking delete button on a group header: immediate deletion — no confirmation dialog
- All dogs in the deleted group are returned to available status in the roster
- Rationale: groups are low-stakes compositions easily recreated; no data is permanently lost

Source: CONTEXT.md D-07, D-14, D-15, D-16

| Element | Copy |
|---------|------|
| Tab label | Groups |
| Primary CTA | + Add Group |
| Empty state heading | No groups yet |
| Empty state body | A group will be created for you automatically. Drag dogs from the roster to get started. |
| Roster panel heading | Available Dogs |

| Group score label | Score: [N] |

| Conflict warning tooltip | [N] conflict[s] in this group |
| Delete group aria-label | Delete [Group Name] |

- No destructive confirmation copy needed for dog removal via × button (low-stakes, reversible)

## States Reference

### Roster Panel States

| State | Visual |
|-------|--------|
| Available dog | Normal opacity, cursor-grab on drag handle, hover:bg-slate-100 |
| Dog in a group | opacity-60, bg-slate-100, cursor-not-allowed, "in [Group Name]" label shown |
| No active dogs | "No active dogs. Add dogs in the Dogs tab." (text-sm text-slate-400, px-3 py-4) |

### Group Panel States

| State | Visual |
|-------|--------|
| Empty group (no dogs) | Dashed drop zone, "Drop a dog here" hint text (text-sm text-slate-400) |
| Group with dogs, no conflicts | Dog mini cards, score badge (green if ≥80), no warning icon |
| Group with dogs, conflicts exist | Dog mini cards, score badge (red if <50), ⚠ icon, red SVG lines between conflicting pairs |
| Group name being edited | Inline input replaces name text, same font/size |
| Drag-over active group | Drop zone border changes to border-primary (slate-900) to indicate valid target |

### Score Badge Color

| Score | Badge classes |
|-------|--------------|
| 80–100 | bg-green-100 text-green-700 |
| 50–79 | bg-yellow-100 text-yellow-700 |
| 0–49 | bg-red-100 text-red-700 |
