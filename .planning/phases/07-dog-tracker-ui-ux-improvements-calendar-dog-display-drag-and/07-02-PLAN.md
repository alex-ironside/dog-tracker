# Plan 07-02: Group Builder DnD Fix

## Goal

Fix the duplicate draggable ID bug in the group builder. When a dog appears in multiple groups, all its MiniDogCards share the same `useDraggable({ id: dog.id })`, causing dnd-kit to treat them as one element. Change the draggable ID to `${groupId}-${dog.id}` so each card is uniquely identified.

## Context

**Bug:** `MiniDogCard.tsx` line 14: `useDraggable({ id: dogId })`. When dog "Rex" is in Group A and Group B, both cards register with the same draggable ID. Dragging one visually drags both.

**Key files:**
- `src/components/MiniDogCard.tsx` — `useDraggable({ id: dogId })` on line 14. Needs `groupId` prop to build unique ID.
- `src/components/GroupPanel.tsx` — renders MiniDogCards on line 133. Has `group.id` in scope. Needs to pass `groupId` prop.
- `src/components/GroupBuilder.tsx` — `handleDragEnd` on line 167 treats `event.active.id` as a plain `dogId`. Must parse `${groupId}-${dogId}` format or use `data` payload.

**Existing tests:**
- `src/components/GroupBuilder.test.tsx` — tests DnD via mocked `onDragEnd` callback pattern. May need updates for the new ID format.
- `src/components/GroupPanel.test.tsx` — tests GroupPanel rendering and interactions.

## Tasks

### Task 1: Update MiniDogCard to accept groupId and use composite draggable ID

**Files:** `src/components/MiniDogCard.tsx`, `src/components/MiniDogCard.test.tsx` (new)

**Action:**

1. **Write tests first** in `src/components/MiniDogCard.test.tsx`:
   - Test: when rendered with `groupId="g1"` and `dogId="d1"`, the `useDraggable` hook receives `id: "g1-d1"`.
   - Test: the `data` payload on useDraggable includes `{ dogId, groupId }` for downstream consumption.
   - Since testing dnd-kit hook internals is hard, a practical approach: mock `@dnd-kit/core` and capture the argument to `useDraggable`. Assert `id === 'g1-d1'` and `data.dogId === 'd1'` and `data.groupId === 'g1'`.

2. **Update `MiniDogCardProps`** — add `groupId: string`.

3. **Change `useDraggable` call** from:
   ```ts
   useDraggable({ id: dogId })
   ```
   to:
   ```ts
   useDraggable({
     id: `${groupId}-${dogId}`,
     data: { dogId, groupId },
   })
   ```

4. **Update the `data` attribute** so `onDragEnd` consumers can extract `dogId` from `event.active.data.current.dogId` instead of parsing the ID string.

### Task 2: Thread groupId through GroupPanel and update GroupBuilder onDragEnd

**Files:** `src/components/GroupPanel.tsx`, `src/components/GroupBuilder.tsx`, `src/components/GroupBuilder.test.tsx` (update)

**Action:**

1. **Write/update tests first:**
   - In `GroupBuilder.test.tsx`, update any test that simulates `onDragEnd` with `active.id = dogId` to instead use `active.id = '${groupId}-${dogId}'` and `active.data.current = { dogId, groupId }`.
   - Add test: when dog is dragged from one group to roster, `removeDogFromGroup` is called with the correct `dogId` (not the composite ID).
   - Add test: when dog is dragged from roster to a group, `addDogToGroup` is called with the group's droppable ID and the plain `dogId`.

2. **Update `GroupPanel.tsx`** — pass `groupId={group.id}` to each `MiniDogCard`:
   ```tsx
   <MiniDogCard
     groupId={group.id}
     dogId={dog.id}
     dogName={dog.name}
     onRemove={() => onRemoveDog(dog.id)}
   />
   ```

3. **Update `GroupBuilder.tsx` `handleDragEnd`** (line 167-187):
   - Extract dogId from the `data` payload instead of using `active.id` directly:
     ```ts
     function handleDragEnd(event: DragEndEvent) {
       setActiveDragId(null)
       const over = event.over
       if (!over) return

       // Extract dogId from data payload (set by MiniDogCard and RosterRow)
       const dogId = (event.active.data.current?.dogId as string) ?? (event.active.id as string)
       const overId = over.id as string

       if (overId === 'roster') {
         for (const group of walkGroups) {
           if (group.dogIds.includes(dogId)) {
             removeDogFromGroup(group.id, dogId)
           }
         }
       } else {
         addDogToGroup(overId, dogId)
       }
     }
     ```
   - The fallback `?? (event.active.id as string)` handles RosterRow cards which don't have `groupId` context (they're in the roster, not a group). Check `RosterRow` — if it uses `useDraggable({ id: dog.id })`, it should also set `data: { dogId: dog.id }` for consistency, but since roster cards are unique (one per dog), the plain ID still works.

4. **Update `handleDragStart`** — the `activeDragId` is used for `DragOverlay` display. Currently it's set to `event.active.id`. For group cards, that's now `${groupId}-${dogId}`. The overlay finds the dog by ID to show its name. Update to extract dogId:
   ```ts
   function handleDragStart(event: DragStartEvent) {
     const dogId = (event.active.data.current?.dogId as string) ?? (event.active.id as string)
     setActiveDragId(dogId)
   }
   ```

5. **Check `RosterRow`** — verify it also sets `data: { dogId: dog.id }` on its `useDraggable` call for consistency. If not, add it.

**Details:** The key insight is that `onDragEnd` should read `dogId` from `event.active.data.current.dogId` rather than parsing the composite string ID. This is cleaner and future-proof. The composite ID (`${groupId}-${dog.id}`) ensures dnd-kit registers unique draggable nodes, while the `data` payload carries the semantic information.
