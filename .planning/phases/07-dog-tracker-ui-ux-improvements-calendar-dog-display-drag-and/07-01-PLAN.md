# Plan 07-01: Calendar Improvements

## Goal

Enhance the calendar view so scheduled slots show individual dog names as pills, indicate when a dog has multiple walks on the same day, let the user highlight/spotlight a specific dog across all slots, remove the fixed 600px height so the grid grows naturally, and add an hour-range selector to narrow visible hours.

## Context

**Key files:**
- `src/components/ScheduledGroupCard.tsx` — currently shows group name + dog count + score badge. Needs dog name pills added below the existing row.
- `src/components/CalendarSlot.tsx` — renders each slot cell, passes `group.dogIds` to ScheduledGroupCard but only as `dogCount`. Needs to pass dog names and multi-walk data.
- `src/components/CalendarScheduler.tsx` — top-level calendar container, has `h-[600px]` fixed height on the flex wrapper (line 131). Needs hour-range state + dog highlight state.
- `src/components/WeekCalendar.tsx` — grid renderer, `HOURS` drives row count, has `overflow: auto` (line 68). Needs to accept filtered hours + highlight dog ID.
- `src/lib/calendarUtils.ts` — exports `HOURS = [7,8,...,19]` (13 hours). The hour-range selector filters this array.

**Types:** `WalkGroup.dogIds: string[]`, `Dog { id, name, ... }`, `WalkSession { groupId, slot: TimeSlot }`, `TimeSlot { dayOfWeek, hour, minute }`.

## Tasks

### Task 1: Add dog name pills to ScheduledGroupCard + multi-walk indicator

**Files:** `src/components/ScheduledGroupCard.tsx`, `src/components/CalendarSlot.tsx`, `src/components/ScheduledGroupCard.test.tsx` (new)

**Action:**

1. **Write tests first** in `src/components/ScheduledGroupCard.test.tsx`:
   - Test: renders dog name pills when `dogNames` prop is provided (e.g. `["Rex", "Luna"]` renders two pill spans with those names).
   - Test: renders multi-walk badge `x2` on a pill when `multiWalkCounts` has `count > 1` for that dog.
   - Test: does NOT render multi-walk badge when count is 1.
   - Test: highlights a pill when `highlightDogId` matches that dog's ID.

2. **Update `ScheduledGroupCard` props** — add:
   ```ts
   dogNames: string[]       // ordered list of dog names in the group
   dogIds: string[]          // parallel array of dog IDs (same order as dogNames)
   multiWalkCounts: Map<string, number>  // dogId -> count of walks that day (only entries > 1 matter)
   highlightDogId?: string | null        // if set, matching pill gets highlight styling
   ```

3. **Render dog pills** below the existing first row (group name + score + conflict). Add a second `<div>` with `className="flex flex-wrap gap-1 mt-1"`. For each dog, render:
   ```tsx
   <span
     key={dogId}
     className={cn(
       'inline-flex items-center gap-0.5 px-1.5 py-0.5 rounded-full text-[10px] font-medium bg-background border border-border',
       highlightDogId === dogId && 'ring-2 ring-primary bg-primary/10'
     )}
   >
     {dogName}
     {(multiWalkCounts.get(dogId) ?? 0) > 1 && (
       <span className="text-primary font-bold" title={`${multiWalkCounts.get(dogId)} walks today`}>
         x{multiWalkCounts.get(dogId)}
       </span>
     )}
   </span>
   ```

4. **Update `CalendarSlot`** to compute and pass the new props:
   - It already has `group` (WalkGroup) and `dogs` (Dog[]). Resolve `dogNames` by mapping `group.dogIds` to `dogs.find(d => d.id === id)?.name ?? '?'`.
   - Receive `multiWalkCounts` and `highlightDogId` as new props on `CalendarSlotProps` (passed from WeekCalendar).
   - Pass `dogNames`, `dogIds: group.dogIds`, `multiWalkCounts`, and `highlightDogId` to `ScheduledGroupCard`.

5. **Update `WeekCalendar`** to:
   - Accept new props: `multiWalkCountsByDay: Map<number, Map<string, number>>` (dayOfWeek -> dogId -> count) and `highlightDogId: string | null`.
   - Pass `multiWalkCounts={multiWalkCountsByDay.get(dayOfWeek) ?? new Map()}` and `highlightDogId` to each `CalendarSlot`.

6. **Compute `multiWalkCountsByDay` in `CalendarScheduler`** with a `useMemo`:
   - Iterate `walkSessions`. For each session, find the group, get its `dogIds`. For each dogId, increment `countsByDay.get(session.slot.dayOfWeek).get(dogId)`.
   - Return `Map<number, Map<string, number>>`.
   - Pass to `WeekCalendar`.

**Details:** The card layout changes from a single row to a two-row layout. The first row stays as-is (group name, score, conflict icon, Log, X). The second row is the pill strip. The card's outer div should change from `flex items-center justify-between` to a vertical flex: `flex flex-col gap-1`. Keep the first row as its own `flex items-center justify-between` div.

### Task 2: Dog highlight control + remove fixed height + hour-range selector

**Files:** `src/components/CalendarScheduler.tsx`, `src/components/WeekCalendar.tsx`, `src/components/CalendarSlot.tsx`, `src/components/CalendarScheduler.test.tsx` (update)

**Action:**

1. **Write/update tests first:**
   - In `CalendarScheduler.test.tsx`, add:
     - Test: renders a dog highlight dropdown/select with all active dog names.
     - Test: selecting a dog sets highlightDogId state (verify via pill ring class or slot highlight class).
     - Test: hour-range selectors render with default 8–20 values.
     - Test: changing start hour to 10 hides the 8:00 and 9:00 rows.
   - Verify existing tests still pass after layout changes.

2. **Dog highlight control** in `CalendarScheduler`:
   - Add state: `const [highlightDogId, setHighlightDogId] = useState<string | null>(null)`.
   - Render a `<select>` above the calendar grid (between header and the flex container):
     ```tsx
     <div className="flex items-center gap-4 mb-3">
       <label className="text-sm text-muted-foreground">Highlight dog:</label>
       <select
         value={highlightDogId ?? ''}
         onChange={(e) => setHighlightDogId(e.target.value || null)}
         className="text-sm border border-border rounded px-2 py-1 bg-background"
       >
         <option value="">None</option>
         {dogs.filter(d => !d.archived).map(d => (
           <option key={d.id} value={d.id}>{d.name}</option>
         ))}
       </select>
     </div>
     ```
   - Pass `highlightDogId` down to `WeekCalendar`.
   - In `CalendarSlot`, when `highlightDogId` is set and the slot's group contains that dogId, add a highlight class to the slot div: `border-2 border-primary` (in addition to passing `highlightDogId` to `ScheduledGroupCard` for pill glow).

3. **Remove fixed height:**
   - In `CalendarScheduler.tsx` line 131, change `<div className='flex h-[600px] rounded-2xl border border-border overflow-hidden'>` to `<div className='flex rounded-2xl border border-border overflow-hidden'>` (remove `h-[600px]`).
   - In `WeekCalendar.tsx` line 68, change `<div className='overflow-auto flex-1'>` to `<div className='flex-1'>` (remove `overflow-auto`). The page itself scrolls if needed.
   - In `WeekCalendar.tsx` line 43, change `<div className='flex flex-col flex-1 overflow-hidden'>` to `<div className='flex flex-col flex-1'>` (remove `overflow-hidden`).

4. **Hour-range selector:**
   - Add state in `CalendarScheduler`:
     ```ts
     const [startHour, setStartHour] = useState(() => {
       const saved = localStorage.getItem('portfolio:calHours')
       return saved ? JSON.parse(saved).start : 8
     })
     const [endHour, setEndHour] = useState(() => {
       const saved = localStorage.getItem('portfolio:calHours')
       return saved ? JSON.parse(saved).end : 20
     })
     ```
   - Persist on change:
     ```ts
     useEffect(() => {
       localStorage.setItem('portfolio:calHours', JSON.stringify({ start: startHour, end: endHour }))
     }, [startHour, endHour])
     ```
   - Compute filtered hours: `const filteredHours = HOURS.filter(h => h >= startHour && h <= endHour)`.
   - Render two `<select>` elements next to the dog highlight control:
     ```tsx
     <label className="text-sm text-muted-foreground">Hours:</label>
     <select value={startHour} onChange={e => setStartHour(Number(e.target.value))} className="text-sm border border-border rounded px-2 py-1 bg-background">
       {HOURS.map(h => <option key={h} value={h}>{String(h).padStart(2,'0')}:00</option>)}
     </select>
     <span className="text-muted-foreground">-</span>
     <select value={endHour} onChange={e => setEndHour(Number(e.target.value))} className="text-sm border border-border rounded px-2 py-1 bg-background">
       {HOURS.map(h => <option key={h} value={h}>{String(h).padStart(2,'0')}:00</option>)}
     </select>
     ```
   - Pass `filteredHours` to `WeekCalendar` as a new prop `hours: number[]`.
   - In `WeekCalendar`, replace `HOURS.map(...)` with `hours.map(...)` in the grid rendering. Update `gridTemplateRows` to use `hours.length` instead of hardcoded 13: `` `40px repeat(${hours.length}, 64px)` ``.

**Details:** Import `HOURS` from `calendarUtils` in `CalendarScheduler` for the select options. The `WeekCalendar` no longer imports `HOURS` directly for rendering — it uses the `hours` prop. Keep `HOURS` in calendarUtils as the full range source.
