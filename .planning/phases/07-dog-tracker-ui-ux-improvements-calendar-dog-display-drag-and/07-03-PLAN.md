# Plan 07-03: Enter-to-Save on DogPanel Inputs

## Goal

Add `onKeyDown` Enter-to-save behavior on the Name, Breed, and Age inputs in `DogPanel.tsx`. Pressing Enter on any of these inputs calls `handleSave()`. The Notes textarea is excluded — Enter inserts a newline there (standard behavior).

## Context

**Key file:** `src/components/DogPanel.tsx`
- `handleSave()` defined on line 82 — validates name, builds payload, calls store `addDog` or `updateDog`, closes panel.
- Name input: line 171-183 — `<Input>` component, no `onKeyDown`.
- Breed input: line 197-202 — `<Input>` component, no `onKeyDown`.
- Age input: line 208-214 — `<Input type="number">`, no `onKeyDown`.
- Notes textarea: line 226-238 — `<textarea>`, no `onKeyDown`. Must NOT get Enter-to-save.
- The GroupBuilder quick-add input already handles Enter (line 78 in GroupBuilder.tsx) — already verified, no changes needed.

**Existing test:** `src/components/DogPanel.test.tsx` — tests rendering, save behavior. Needs new keyboard tests.

## Tasks

### Task 1: Add onKeyDown Enter-to-save to DogPanel inputs + tests

**Files:** `src/components/DogPanel.tsx`, `src/components/DogPanel.test.tsx` (update)

**Action:**

1. **Write tests first** in `src/components/DogPanel.test.tsx` — add these test cases:
   - Test: pressing Enter on Name input calls save (dog is added/updated). Setup: render DogPanel open, type a name into the Name input, press Enter via `userEvent.keyboard('{Enter}')` or `fireEvent.keyDown(input, { key: 'Enter' })`. Assert the panel closes (onOpenChange called with false) and the store has the new dog.
   - Test: pressing Enter on Breed input triggers save. Setup: fill name (required) + breed, press Enter on breed input. Assert save completes.
   - Test: pressing Enter on Age input triggers save. Setup: fill name + age, press Enter on age input. Assert save completes.
   - Test: pressing Enter on Notes textarea does NOT trigger save. Setup: fill name + notes, press Enter in textarea. Assert panel is still open and no dog was added.
   - Test: pressing Shift+Enter on Name input does NOT trigger save. Assert panel stays open.
   - Test: pressing Enter on Name input with empty name shows validation error (name required), does NOT close panel.

2. **Create a shared handler function** inside `DogPanel` component:
   ```ts
   function handleInputKeyDown(e: React.KeyboardEvent<HTMLInputElement>) {
     if (e.key === 'Enter' && !e.shiftKey) {
       e.preventDefault()
       handleSave()
     }
   }
   ```

3. **Add `onKeyDown={handleInputKeyDown}`** to the three `<Input>` components:

   Name input (around line 171):
   ```tsx
   <Input
     id={nameId}
     placeholder="e.g. Rex"
     value={name}
     onChange={(e) => {
       setName(e.target.value)
       if (nameError && e.target.value.trim()) setNameError(false)
     }}
     onKeyDown={handleInputKeyDown}
     className={cn(
       'mt-1',
       nameError && 'border-destructive focus-visible:ring-destructive'
     )}
   />
   ```

   Breed input (around line 199):
   ```tsx
   <Input
     id={breedId}
     placeholder="e.g. Labrador"
     value={breed}
     onChange={(e) => setBreed(e.target.value)}
     onKeyDown={handleInputKeyDown}
     className="mt-1"
   />
   ```

   Age input (around line 210):
   ```tsx
   <Input
     id={ageId}
     type="number"
     min={0}
     placeholder="e.g. 3"
     value={age}
     onChange={(e) => setAge(e.target.value)}
     onKeyDown={handleInputKeyDown}
     className="mt-1"
   />
   ```

4. **Do NOT add `onKeyDown` to the Notes `<textarea>`** (line 226). Leave it as-is. Enter in a textarea naturally inserts a newline — this is the desired behavior.

**Details:** The `handleInputKeyDown` function is simple and self-contained. It calls the existing `handleSave()` which already handles validation (empty name check) and cleanup (resetAndClose). The `e.preventDefault()` stops the Enter key from triggering any default form behavior. The `!e.shiftKey` guard allows Shift+Enter to pass through without saving (future-proofing for any paste scenarios).
